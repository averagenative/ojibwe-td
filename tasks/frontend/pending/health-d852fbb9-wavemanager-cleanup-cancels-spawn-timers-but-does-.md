## Finding

**Category:** MISSING_CLEANUP
**Severity:** low
**Location:** `systems/WaveManager.ts:519`

cleanup() destroys Phaser timer events and clears _activeWaves but does not remove the wave-complete EventEmitter listeners registered by GameScene via waveManager.on('wave-complete', ...). Because WaveManager is recreated every create() call the old instance is garbage-collected without issue today, but if any code held a reference across scene restarts the stale listener would fire on the next wave. Fix: add this.removeAllListeners() at the end of cleanup() as a defensive guard.

## Finding

**Category:** STUB
**Severity:** low
**Location:** `systems/AmbientVFX.ts:785`

_randomTile calls this._rng(arr.length), using the array length as the RNG salt. posHash is deterministic per (seed, elapsed_time_bucket, salt), so if _waterEdgeTiles and _buildableTiles have equal length, shimmer and vine-glow effects always pick the same tile index in a given frame tick. This causes visible spatial correlation between otherwise-independent visual effects. Fix: pass a unique per-effect salt constant instead of arr.length.

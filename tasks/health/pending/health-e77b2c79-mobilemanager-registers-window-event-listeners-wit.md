## Finding

**Category:** MISSING_CLEANUP
**Severity:** medium
**Location:** `systems/MobileManager.ts:26`

The constructor registers resize and orientationchange handlers via anonymous arrow functions (lines 26-27). Since no reference to the closure is stored, these listeners can never be removed. The singleton pattern prevents duplicate listeners in production, but in test environments (where the module is re-imported) or if _instance is ever reset, duplicate listeners accumulate. Fix: store the bound callbacks in private fields and expose a destroy() method that calls window.removeEventListener.

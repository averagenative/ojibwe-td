# iOS Build Guide

How to build and deploy Ojibwe TD to your iPhone/iPad.

## Prerequisites

- macOS with Xcode 16+ installed
- An Apple Developer account (free works for personal device testing)
- iPhone/iPad connected via USB (or wireless pairing set up in Xcode)
- Node 18+
- See [ios-build-dependencies.md](ios-build-dependencies.md) for version details

## Quick Reference

```bash
cd game

# One-liner: build web assets + sync to iOS + open Xcode
npm run build:ios
```

Then in Xcode, select your device and hit **Play** (Cmd+R).

That's it for most builds. Read on for details.

---

## Step-by-Step

### 1. Install dependencies (first time or after pulling)

```bash
cd game
npm ci          # use ci, not install, to respect lockfile
```

### 2. Build the web assets

```bash
npm run build
```

This runs TypeScript compilation and Vite bundling. Output goes to `game/dist/`.

### 3. Sync web assets into the iOS project

```bash
npx cap sync ios
```

This copies `dist/` into `ios/App/App/public/` and updates native plugins. **You must run this every time you change web code** — Xcode's Play button only compiles the Swift wrapper, it does NOT rebuild the web game.

### 4. Open Xcode

```bash
npx cap open ios
```

Or manually: open `game/ios/App/App.xcodeproj`

### 5. Build and run

1. Select your device in the device dropdown (top center of Xcode)
2. If first time: Xcode > Settings > Accounts — sign in with your Apple ID
3. Select the "App" target > Signing & Capabilities > set your Team
4. Hit **Play** (Cmd+R) or Product > Run

### 6. Trust the developer certificate (first time on device)

On your iPhone/iPad: Settings > General > VPN & Device Management > tap your developer profile > Trust.

## Common Workflows

### "I changed game code and want to test on my phone"

```bash
npm run build && npx cap sync ios
```

Then hit Play in Xcode. The web assets are baked in at sync time — you must re-sync.

### "I want to iterate quickly in the browser first"

```bash
npm run dev
```

Open http://localhost:5173 in your browser. Use Chrome DevTools mobile emulation for approximate phone testing. When ready for real device testing, do the full build+sync above.

### "The app on my phone looks outdated"

You probably changed code but forgot to sync. Run:

```bash
npm run build && npx cap sync ios
```

Then rebuild in Xcode. The Play button in Xcode **only** compiles Swift — it does NOT rebuild your TypeScript/web code.

### "Xcode shows Swift compile errors"

```
File > Packages > Reset Package Caches
Product > Clean Build Folder (Cmd+Shift+K)
```

Then build again. See [ios-build-dependencies.md](ios-build-dependencies.md) for version-specific issues.

### "I want to test on the simulator instead of a real device"

Select a simulator from the Xcode device dropdown (e.g., "iPhone 16 Pro") and hit Play. No developer certificate needed for simulators.

## Pipeline Summary

```
TypeScript source
    |
    v
npm run build        -->  game/dist/          (Vite bundle)
    |
    v
npx cap sync ios     -->  ios/App/App/public/ (copied into native project)
    |
    v
Xcode Play (Cmd+R)   -->  device/simulator    (Swift shell + embedded web app)
```

The game is a web app running inside a native WKWebView shell. Capacitor bridges native APIs (haptics, keep-awake) to JavaScript.

## Project Structure

```
game/ios/App/
  App.xcodeproj/       -- Xcode project (Capacitor expects this name)
  App/
    AppDelegate.swift   -- native app lifecycle
    public/             -- web assets (auto-generated by cap sync, do NOT edit)
    Info.plist          -- app metadata, display name "Ojibwe TD"
    Assets.xcassets/    -- app icon
  CapApp-SPM/           -- Swift Package Manager deps (Capacitor framework)
```

**Important:** The Xcode project must be named `App.xcodeproj`. Capacitor hardcodes this path. Do not rename it in Xcode.
